# 보안 정책

[English](../SECURITY.md) | **한국어**

## 지원 버전

| 버전 | 지원 여부 |
|------|----------|
| 최신 | :white_check_mark: |

DiveChecker는 활발히 개발 중이며, 보안 업데이트는 `main` 브랜치의 최신 버전에 적용됩니다.

## 취약점 제보 방법

**보안 취약점은 공개 GitHub 이슈로 제보하지 마세요.**

대신 [GitHub Security Advisories](https://github.com/kernalix7/DiveChecker/security/advisories/new)를 통해 제보해 주세요.

### 제보 시 포함할 내용

취약점 제보 시 다음 정보를 포함해 주세요:

1. **설명** — 취약점에 대한 명확한 설명
2. **재현 단계** — 문제를 재현하기 위한 상세 단계
3. **영향** — 취약점의 잠재적 영향
4. **영향받는 구성요소** — DiveChecker의 어느 부분이 영향받는지 (앱, 펌웨어, 하드웨어)
5. **환경** — OS, Flutter 버전, 펌웨어 버전, 디바이스 모델

### 대응 일정

- **접수 확인** — 제보 후 48시간 이내
- **초기 평가** — 7일 이내
- **수정 및 공개** — 제보자와 협의하여 진행; 심각한 이슈는 통상 30일 이내

### 범위 (In Scope)

보안 제보 대상 범위:

- ECDSA 디바이스 인증 우회 또는 키 추출
- USB MIDI SysEx 프로토콜 인젝션 또는 버퍼 오버플로우
- 펌웨어 메모리 손상 또는 코드 실행
- Flutter 앱 데이터 유출 (세션 데이터, 디바이스 키)
- 크로스플랫폼 데이터베이스 (SQLite/IndexedDB) 인젝션
- RP2350 OTP 키 저장소 취약점
- 불안전한 펌웨어 업데이트 메커니즘
- Bluetooth/USB 통신 가로채기

### 범위 외 (Out of Scope)

- 사용자 머신에 물리적 접근이 필요한 버그
- 소셜 엔지니어링 공격
- 서드파티 의존성의 문제 (업스트림에 제보해 주시되, 알려주시면 감사합니다)

## 보안 모범 사례

DiveChecker가 따르는 보안 관행:

- **ECDSA P-256 디바이스 인증** — 정품 하드웨어의 암호학적 검증
- **OTP 키 저장** — 프로덕션 키는 RP2350 일회성 프로그래밍 메모리에 저장
- **펌웨어 동적 메모리 미사용** — 고정 크기 버퍼로 힙 기반 공격 방지
- **입력 검증** — 모든 SysEx 명령은 처리 전 검증
- **PIN 보호 설정** — 디바이스 설정 변경 시 PIN 인증 필요
- **네트워크 통신 없음** — 순수 USB 연결, 클라우드/인터넷 의존성 없음

## 감사의 말

책임감 있는 취약점 공개에 힘써주시는 보안 연구 커뮤니티에 감사드립니다. 유효한 보안 이슈를 제보해 주신 분은 (동의 시) 릴리스 노트에서 감사를 표합니다.
